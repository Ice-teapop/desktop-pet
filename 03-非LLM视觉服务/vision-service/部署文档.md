## 二、基础部署

```bash
# 1. 取代码、装依赖（建议用虚拟环境）
cd vision-service
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 2. 装 tesseract（Ubuntu/Debian）
sudo apt-get update
sudo apt-get install -y tesseract-ocr tesseract-ocr-eng tesseract-ocr-chi-sim
#   读中文必须装 chi-sim，并把 config/config.yaml 的 ocr.lang 改为 "chi_sim+eng"

# 3. 配 token（务必用环境变量，不要把真实 token 写进 config.yaml）
export DESKPET_VISION_TOKEN="你的密钥"

# 4. 启动
uvicorn src.main:app --host 0.0.0.0 --port 8800 --workers 1

# 5. 验证
curl http://localhost:8800/v1/health
```

生产环境建议在前面挂一层反向代理（nginx 等）做 **HTTPS/TLS** —— 接口契约要求全程 HTTPS，
反向代理同时也可以再设一道请求体大小上限。

### 用 systemd 托管（生产推荐）

项目自带两份配置在 `deploy/` 目录：
- `deploy/deskpet-vision.service` —— systemd 服务单元
- `deploy/deskpet-vision.env.example` —— 环境变量模板

在服务器上（root）：

```bash
# 1. 建专用非 root 用户（带 home，供 pix2tex / torch 缓存模型权重用）
useradd -r -m -d /opt/deskpet/home -s /usr/sbin/nologin deskpet

# 2. 放代码 + 建虚拟环境
mkdir -p /opt/deskpet
cp -r vision-service /opt/deskpet/
cd /opt/deskpet/vision-service
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
chown -R deskpet:deskpet /opt/deskpet

# 3. 配 token（真实 token 不进 git）
cp deploy/deskpet-vision.env.example /etc/deskpet-vision.env
#   编辑 /etc/deskpet-vision.env 填入真实 token（生成： openssl rand -hex 32）
chmod 600 /etc/deskpet-vision.env

# 4. 装并启动服务
cp deploy/deskpet-vision.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable --now deskpet-vision

# 5. 看状态与日志
systemctl status deskpet-vision
journalctl -u deskpet-vision -f
```

服务单元已内置：单 worker、PyTorch / BLAS 线程限制（适配 2GB / 1 vCPU）、失败自动重启、
非 root 用户运行、`NoNewPrivileges` / `PrivateTmp` / `ProtectSystem` 基础加固。token 通过
`EnvironmentFile` 注入，不写进单元文件、不进 git。

> 注：systemd 单元文件无法在开发环境实测，需在服务器上 `systemctl status` 确认。
> 若启动失败，先看 `journalctl -u deskpet-vision` —— 常见原因：未配 token（启动安全自检会拒绝）、
> 路径不对、依赖没装全。

## 三、在 2GB 小机器上部署（重点）

2GB + pix2tex 是「紧、但能弄」。**不启用公式识别**时 2GB 很宽裕；**启用 pix2tex**
时按下面几条做，把「会 OOM 崩」变成「偶尔慢、不崩」。

### 3.1 加 swap 文件（最重要，免费，几分钟）

pix2tex 是低频重操作（只有区域里真出现公式才触发）。加 swap 后，内存峰值来了就
「稍微慢一下」走 swap，而不是被 OOM 杀掉。

```bash
# 创建 2GB swap 文件（fallocate 不可用时用 dd 见下）
sudo fallocate -l 2G /swapfile
# 备选： sudo dd if=/dev/zero of=/swapfile bs=1M count=2048
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 开机自动挂载
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# 可选：降低 swappiness，让系统只在真有内存压力时才用 swap
sudo sysctl vm.swappiness=10
echo 'vm.swappiness=10' | sudo tee -a /etc/sysctl.conf

# 确认
free -h
```

### 3.2 只开单 worker

pix2tex 模型每个 worker 各占一份内存，2GB 也只够开 1 个：

```bash
uvicorn src.main:app --host 0.0.0.0 --port 8800 --workers 1
```

（服务里的 OCR / 公式引擎已是模块级单例，单 worker 内请求间复用，不会重复加载。）

### 3.3 限制 PyTorch 线程

启动服务前设这几个环境变量，限制 PyTorch / BLAS 的线程数 —— 既省内存，也避免
单 vCPU 上线程互相抢占：

```bash
export OMP_NUM_THREADS=1
export MKL_NUM_THREADS=1
export OPENBLAS_NUM_THREADS=1
```

把它们和 `DESKPET_VISION_TOKEN` 一起写进 systemd 服务文件或启动脚本。

### 3.4 内存预期

| 组件 | 大致常驻内存 |
|---|---|
| 操作系统 | ~300–400 MB |
| 基础服务（FastAPI + tesseract + Pillow） | ~150–250 MB |
| pix2tex（完整 PyTorch CPU，模型已加载） | ~800 MB – 1.2 GB |
| **合计：启用 pix2tex** | ~1.3–1.8 GB → 2GB 偏紧，靠 swap 兜底 |
| **合计：不启用 pix2tex** | ~0.5–0.7 GB → 2GB 很宽裕 |

结论：不用公式识别，2GB 直接够；用公式识别，2GB + swap + 单 worker + 线程限制能跑，
就是不会快（共享 CPU + 偶尔走 swap，一个公式可能要几秒）。

### 3.5 公式识别（pix2tex）按需启用

```bash
# 要启用公式识别才装；不装则公式区域自动降级为文本，服务照常
pip install pix2tex
```

注意：

- `pix2tex` 含 PyTorch，安装包较大（~1GB+），首次调用会下载模型权重。
- 不装 pix2tex：`Pix2TexEngine.available()` 返回 False，公式区域优雅降级为文本，服务不受影响。
- pix2tex 的实际推理路径未在开发环境实测（依赖过重），**首次部署后请按第六节验证公式路径**。

## 四、更省内存的方案：ONNX（规划中，暂未实现）

如果 2GB + 完整 PyTorch 还是觉得紧，更彻底的省内存路线是用 **ONNX 跑 pix2tex** ——
`onnxruntime` 比完整 PyTorch 轻很多。

**现状（如实说明）**：代码里目前**只有**基于完整 PyTorch 的 `Pix2TexEngine`，
**还没有** ONNX 引擎。要走这条路需要：

1. 把 pix2tex 模型导出成 ONNX 格式；
2. 新增一个 `OnnxFormulaEngine`（实现 `pipeline/formula/base.py` 的 `FormulaEngine` 抽象，用 onnxruntime 跑）；
3. 把 `config/config.yaml` 的 `formula.engine` 切到新引擎。

架构上是支持的（公式引擎本就可插拔），但这部分属于待实现项 —— 需要时再单独做一步。

## 五、配置项速查

配置在 `config/config.yaml`，敏感项用环境变量覆盖。

| 项 | 位置 | 说明 |
|---|---|---|
| bearer_token | env `DESKPET_VISION_TOKEN`（优先）/ config | 鉴权 token，务必用环境变量 |
| ocr.lang | config | 读中文设 `"chi_sim+eng"` |
| limits.max_image_bytes / max_image_pixels | config | 请求体 / 像素上限 |
| limits.rate_per_min | config | 按 token 限流 |
| layout.* | config | 版面分析参数 |
| table.* | config | 表格识别参数 |
| formula.engine | config | 公式引擎，默认 `pix2tex` |
| logging.level | config | 日志级别 |

## 六、上线后验证清单

```bash
# 健康检查
curl https://你的域名/v1/health

# 能力查询
curl https://你的域名/v1/capabilities \
  -H "Authorization: Bearer 你的密钥" -H "X-DeskPet-Client: curl/test"

# 提交一帧（任意 PNG）
curl -X POST https://你的域名/v1/extract \
  -H "Authorization: Bearer 你的密钥" -H "X-DeskPet-Client: curl/test" \
  -F "image=@frame.png;type=image/png" \
  -F 'metadata={"region_id":"r1","frame_seq":1,"captured_at":"2026-05-15T00:00:00Z","region_size":{"w":640,"h":320},"content_hash":"p:abc"}'
```

逐项确认：

- [ ] `/v1/health` 返回 `ok: true`
- [ ] `/v1/capabilities` 返回识别器列表
- [ ] 发文字图 → 返回 `type=text` 区块
- [ ] 发表格图 → 返回 `type=table`、`target=table-json` 区块
- [ ] （若装了 pix2tex）发公式图 → 返回 `type=formula`、`target=latex` 区块
- [ ] （若未装 pix2tex）公式图不报错，降级为文本
- [ ] `free -h` 看内存余量，必要时确认 swap 已生效
- [ ] 服务日志只记指标（region_id / 延迟 / 区块数），不含画面内容
